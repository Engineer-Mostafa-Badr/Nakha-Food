# Nakha-Food

Nakha-Food هو تطبيق يساعد العملاء في الوصول إلى مزوِّدي الخدمات والمنتجات بسرعة وسلاسة، من خلال شاشة واحدة تجمع المحادثات، الطلبات، المحفظة، والمفضلة. نُحافظ على تجربة انسيابية عبر تعدد اللغات (العربية، الإنجليزية، الأردية) وتصميم يستجيب لكل المقاسات.

## لمحة سريعة

- تسجيل أكثر من طريقة (رقم هاتف أو إيميل) مع التحقق عبر رمز.
- دردشة مباشرة داخل التطبيق مع المزوِّد أو الدعم.
- صفحة شخصية تعرض بيانات العميل، المحفظة، حال الطلبات، والمفضلة.
- إشعارات فورية وذكاء في التوصيل بفضل Firebase وPusher.
- استجابة لتحديثات التطبيق تلقائيًا (رقم الإصدار يُستخرج من PackageInfo).

## كيف يتم ترتيب المشروع

```
lib/
├── config/          # إعدادات عامة: الثيم، التوجيه، Firebase
├── core/
│   ├── api/         # التواصل مع الخوادم
│   ├── components/  # أزرار، شاشات، عناصر متكررة
│   ├── cubit/       # إدارة الحالة العامة
│   ├── extensions/  # وظائف مساعدة سهلة الاستخدام
│   ├── res/         # الصور والرموز المشتركة
│   ├── services/    # إشعارات Firebase وPusher وخدمات مساعدة أخرى
│   ├── storage/     # تخزين آمن للبيانات مثل التوكن والخيارات
│   ├── usecase/     # قواعد للعمل المشترك بين الميزات
│   └── utils/       # مساعدات سريعة: التحقق، تحجيم الشاشة، التحويلات
└── features/        # كل ميزة مخططة بشكل مستقل
    ├── auth/        # الدخول والتسجيل والتحقق
    ├── chat/        # صفحة المحادثات والرسائل
    ├── home/        # الشاشة الرئيسية للمحتوى والمنتجات
    ├── profile/     # الملف الشخصي وتعديل البيانات
    ├── orders/      # عرض الطلبات وتفاصيلها
    ├── providers/   # إدارة المزوِّدين ومحتواهم
    ├── favourite/   # تصفح وحفظ المفضلات
    ├── wallet/      # المحفظة والتعاملات المالية
    ├── notifications/ # كل ما يتعلق بالإشعارات
    ├── support/     # دعم العملاء وأسئلة متكررة
    └── onboarding/  # شاشات شرح التطبيق للمرة الأولى
```

كل ميزة تحتوي على ثلاثة أقسام: البيانات (الاتصال بالخادم)، المنطق (الكيانات وقواعد العمل)، والواجهة (الصفحات والأزرار التي يراها المستخدم).

### الأصول (Assets)

- صور وأيقونات مختلفة الدقة داخل `assets/images/`.
- خطوط عائلة Tajawal بأوزان متنوعة من `assets/fonts/`.
- ملفات الترجمة (`assets/translation/`) جاهزة لكل لغة مدعومة.

## كيف تجهّز المشروع

1. حمّل الحزم:
   ```bash
   flutter pub get
   ```
2. إذا عدّلت نماذج أو إشعارات، شغّل:
   ```bash
   flutter pub run build_runner build --delete-conflicting-outputs
   ```
3. لو استبدلت شعارات أو شاشة بداية:
   ```bash
   dart run flutter_launcher_icons:main
   dart run flutter_native_splash:create
   ```
4. شغّل التطبيق:
   ```bash
   flutter run
   ```

## ملاحظات لطيفة

- التطبيق يحدد على أي شاشة يفتح (تعريف، دخول، أو الصفحة الرئيسية) حسب حالة المستخدم المخزنة.
- جميع البيانات الحساسة محفوظة في `MainSecureStorage` لضمان أمان التوكنات والمفاتيح.
- بنية Clean Architecture مع BLoC تعني أن كل ميزة منفصلة، وبالتالي تعديل التصميم أو إضافة شاشة لا يؤثر على باقي التطبيق.
